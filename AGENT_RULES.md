# قوانین کار

## اختیار توسعه
- اختیار کامل توسعه به سمت سرعت و پایداری به تو داده شده
- هیچ سؤالی نپرس، هیچ توضیحی قبل از شروع نده
- مستقیم شروع کن

## محدودیت روزانه و نحوه کار
- من روزانه ۱۰۰ کردیت دارم — هر session ممکنه وسط کار قطع بشه
- اول کار، کل پروژه رو آنالیز کن و یه فایل PLAN.md بساز با لیست همه استپ‌ها
- هر استپ دقیقاً ۴ تا ۶ کار داشته باشه
- هر بار که شروع میکنی، اول PLAN.md رو بخون، ببین کدوم استپ‌ها تکمیل شدن، از اولین استپ ناتمام شروع کن
- هیچوقت منتظر تأیید من نمون برای شروع استپ جاری — فقط بعد از تموم شدن هر استپ صبر کن

## قانون طلایی — صفر توضیح اضافه
هر خطی که با این‌ها شروع میشه را ننویس:
- "Let me look at..." / "My theory is..." / "OK I'll stop..." / "Hmm..." / "Now let me..." / "Let me think..."
- هر جمله‌ای که فقط توضیح میده بدون اینکه ابزار صدا بزنه

**قانون:** اگر میخوای چیزی رو بفهمی، مستقیم ابزار صدا بزن. فکر کردن رو ننویس.

## قانون باگ و مشکلات — مهم
وقتی وسط یه استپ به باگ یا مشکل پیچیده برخوردی:
- **اصلاحش نکن** — وقت و اعتبار کافی نیست
- فقط یادداشتش کن در PLAN.md زیر بخش `## 🐛 باگ‌های یافته‌شده`
- استپ جاری رو با همون هدف اصلیش تموم کن
- در PLAN.md یه استپ جدید با اولویت بالا **قبل از استپ بعدی** اضافه کن برای رفع اون باگ
- در گزارش پایان استپ، باگ رو با یک خط ذکر کن

فرمت ثبت باگ در PLAN.md:
```
## 🐛 باگ‌های یافته‌شده
- [استپ N] نام باگ: توضیح یک‌خطی — → استپ N+1 رفع میشه
```

## فرمت PLAN.md
```
- [ ] استپ ۱ — عنوان
- [x] استپ ۲ — عنوان  ✅ تاریخ
- [ ] استپ ۳ — عنوان (اولویت بالا — رفع باگ از استپ ۲)
```

## آخر هر استپ — اجباری، بدون استثنا، بدون پرسیدن
1. PLAN.md رو آپدیت کن (استپ رو [x] بزن + باگ‌های جدید ثبت کن)
2. تغییرات رو commit و push کن به GitHub — هیچوقت این مرحله رو skip نکن و از من نپرس:
   ```
   git add -A
   git commit -m "step N: [عنوان استپ]"
   git push origin main
   ```
3. گزارش ۳ خطی بده
4. صبر کن من بگم «ادامه» — خودت استپ بعدی رو شروع نکن

## گزارش هر استپ (فقط ۳ تا ۴ خط)
✅ استپ N تموم شد: [یک جمله]
🐛 باگ ثبت‌شده: [اگه بود — یک جمله] یا حذف این خط اگه نبود
📦 push شد: git commit "step N: ..."
⏸ منتظر تأیید برای استپ N+1: [عنوان]

## محدودیت‌های فنی

- هدف: سرعت و پایداری — انتخاب روش فنی با توئه
